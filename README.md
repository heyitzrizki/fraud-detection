# RiskLens AI: Explainable Fraud & Risk Intelligence Platform

RiskLens AI is an MVP fraud risk intelligence platform built with the IEEE-CIS Fraud Detection dataset. Instead of focusing only on fraud classification accuracy, the project demonstrates how machine learning can support operational fraud decisioning through risk scoring, cost-sensitive threshold optimization, explainability, alert prioritization, and analyst-style summaries.

## Project Overview

RiskLens AI is a clean MVP fraud risk analytics project using the IEEE-CIS Fraud Detection dataset. It demonstrates an end-to-end workflow from exploration and feature engineering to model training, threshold optimization, explainability, and a Streamlit dashboard for business review.

## Business Problem

Fraud detection is not just a classification problem. A useful fraud workflow must balance fraud capture, false positives, investigation workload, and business cost. High accuracy can be misleading when fraud is rare, because a model can appear accurate while missing the cases that matter most.

This project answers a practical business question:

Which transactions should be prioritized for fraud investigation, why were they flagged, and what business trade-off happens when the fraud alert threshold changes?

## Dataset

This MVP uses the IEEE-CIS Fraud Detection dataset. The workflow uses `train_transaction.csv` only, placed at:

```text
data/raw/train_transaction.csv
```

The optional `train_identity.csv` file can also be stored in `data/raw/`, but it is not used in the MVP model.

## MVP Features

- Fraud risk scoring
- Time-based validation
- Cost-sensitive threshold optimization
- SHAP explainability
- Alert queue
- Template-based analyst notes
- Streamlit dashboard

## How to Run

Install dependencies:

```bash
pip install -r requirements.txt
```

Run the notebook to train the model and generate outputs:

```bash
jupyter notebook notebooks/01_risklens_ai_mvp.ipynb
```

Launch the dashboard:

```bash
streamlit run app.py
```

## Key Metrics

These values are generated by the notebook and saved to `outputs/metrics_summary.json`.

- PR-AUC: 0.455
- Recommended threshold: 0.30

## Business Interpretation

RiskLens AI is designed to help fraud analysts prioritize which transactions to investigate. The model produces a risk score, the threshold simulator shows the trade-off between fraud capture and alert workload, and the alert queue turns model output into a review-ready format with risk drivers and recommended actions.

## Limitations

- Uses `train_transaction.csv` only
- Identity data is not used yet
- Test data is not used because labels are unavailable
- Cost assumptions are simplified
- Not production-ready
